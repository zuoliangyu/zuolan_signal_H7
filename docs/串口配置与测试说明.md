# UART Configuration and Test Guide

This document covers the UART stack used in this project and how to verify it.

## 1. Hardware Configuration

Two UARTs are enabled:

- `USART1`
  - `TX = PA9`
  - `RX = PA10`
  - `115200 8N1`
  - `RX DMA = DMA1_Stream0`
  - `DMA Request = DMA_REQUEST_USART1_RX`
- `USART2`
  - `TX = PA2`
  - `RX = PA3`
  - `115200 8N1`
  - `RX DMA = DMA1_Stream1`
  - `DMA Request = DMA_REQUEST_USART2_RX`

The `.ioc` mirrors this configuration:

- `USART1_RX -> DMA1_Stream0`
- `USART2_RX -> DMA1_Stream1`
- `USART1 global interrupt`
- `USART2 global interrupt`
- `DMA1_Stream0_IRQn`
- `DMA1_Stream1_IRQn`

## 2. Software Structure

Layered split:

- `Core/dma.*`
  - Enable `DMA1` clock
  - Initialize `DMA1_Stream0/1` NVIC
- `Core/usart.*`
  - Initialize `USART1/USART2`
  - Initialize RX DMA
  - Bind `huartx.hdmarx`
  - Enable `USARTx_IRQn` and `DMA1_Streamx_IRQn`
- `Core/stm32h7xx_it.*`
  - `DMA1_Stream0_IRQHandler()`
  - `DMA1_Stream1_IRQHandler()`
  - `USART1_IRQHandler()`
  - `USART2_IRQHandler()`
- `App/uart/uart.*`
  - `my_printf()` blocking print
  - `UART_Init()` starts `ReceiveToIdle + DMA`
  - `HAL_UARTEx_RxEventCallback()` handles `IDLE / HT / TC`
  - Software ring buffer
  - `uart_proc()` consumes ring buffer

## 3. RX Scheme

- `USART1/USART2` both run `RX DMA + Circular`
- `HAL_UARTEx_ReceiveToIdle_DMA()`
- `HAL_UARTEx_RxEventCallback()` handles:
  - `HAL_UART_RXEVENT_HT` (half)
  - `HAL_UART_RXEVENT_TC` (full)
  - `HAL_UART_RXEVENT_IDLE` (idle line)

Flow:

1. UART bytes enter DMA buffer
2. DMA half/full and idle events fire
3. Callback computes delta vs last position
4. Data pushed into software ring buffer
5. `uart_proc()` drains and processes

## 4. Buffer Design

Per UART:

- DMA RX buffer: `256B`
- Software ring buffer: `1024B`

DMA RX buffers are placed into `.dma_buffer` in `RAM_D2` because:

- Default `.data/.bss` is in `DTCMRAM`
- STM32H743 DMA cannot access `DTCM`

## 5. Current Behavior

`main.c` integration:

1. `MX_USART1_UART_Init()`
2. `MX_USART2_UART_Init()`
3. `UART_Init()`
4. Register `uart_proc()` at `1ms`

Boot behavior:

- `USART1` runs CLI mode and prints prompt `> `
- `USART2` keeps raw echo (fallback)

Send on `USART1`:

```text
help
```

You should see a command list and prompt.

Send on `USART2`:

```text
hello
```

You should get the same echo:

```text
hello
```

## 6. UART FIFO Status

FIFO thresholds are configured but FIFO mode is disabled to keep event timing simple.

## 7. D-Cache Considerations

Current setup:

- `I-Cache` enabled
- `D-Cache` disabled

If `D-Cache` is enabled later, you must either:

1. Place DMA buffers in non-cacheable RAM
2. Invalidate D-Cache before reading DMA buffers

## 8. my_printf

Current function:

```c
int my_printf(UART_HandleTypeDef *huart, const char *format, ...);
```

Notes:

- Uses `vsnprintf`
- Blocking `HAL_UART_Transmit()`
- OK for debug prints, not heavy logs

## 9. Scope

Good for:

- Basic UART RX/TX verification
- Debug prints
- Simple CLI

Not good for:

- High-rate logging
- High-throughput links
- `D-Cache` enabled without coherency handling
