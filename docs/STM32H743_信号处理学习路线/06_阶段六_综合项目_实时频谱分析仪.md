# 第六阶段：综合项目路线

## 项目建议

建议你用 `STM32H743` 做一个“实时频谱分析仪”或“实时采样分析终端”。这个项目对信号处理学习非常合适，因为它把前面所有关键点都串起来了。

## 为什么选这个项目

它覆盖的核心能力非常完整：

- 定时采样
- `ADC + DMA`
- 双缓冲处理
- `Cache` 一致性
- `CMSIS-DSP`
- `FFT`
- 峰值检测
- 实时数据显示或上位机输出

它既能体现 `H743` 的性能优势，也能暴露系统设计上的真实问题。

## 项目分阶段实施建议

### 第 1 步：做稳定采样

目标：

- 单通道稳定采样
- 固定采样率
- 双缓冲搬运

验收：

- 能持续采样数分钟不出错
- 能输出块计数、丢块计数、处理超时计数

### 第 2 步：做基础统计显示

目标：

- 每个 block 输出均值、峰峰值、`RMS`

验收：

- 输入不同幅度信号时，统计量变化符合预期

### 第 3 步：做 FFT 频谱

目标：

- 对每个数据块做窗函数和 `FFT`
- 输出主峰频率和幅值

验收：

- 输入已知频率信号时，主峰位置基本正确
- 你能解释频率分辨率

### 第 4 步：做多模式切换

目标：

- 支持时域模式和频域模式
- 支持不同采样率和不同 `FFT` 点数

验收：

- 切换模式后系统仍稳定
- 参数变化后链路行为可解释

### 第 5 步：做工程化完善

目标：

- 模块分层
- 调试统计
- 参数集中配置
- 错误处理

验收：

- 工程结构清晰
- 能快速定位问题

## 建议项目架构

可以按下面思路拆：

- `adc_capture`
  负责定时触发、采样、`DMA` 缓冲切换
- `buffer_manager`
  负责 block 状态管理、可处理数据块队列
- `signal_process`
  负责均值、`RMS`、滤波、`FFT`
- `ui_or_output`
  负责串口、屏幕或上位机输出
- `monitor`
  负责计时、错误计数、性能统计

## 关键设计点

### 1. 先定义 block 大小

这个决定会同时影响：

- 中断频率
- 算法延迟
- FFT 点数
- 内存占用

### 2. 先定义观测手段

项目里至少保留一个可靠观测口：

- 串口输出统计值
- 上位机接收原始或处理后数据
- 屏幕显示频谱

### 3. 提前考虑缓存策略

在项目早期就规划：

- `DMA buffer` 放哪
- 算法缓冲放哪
- 是否启用 `D-Cache`
- 怎么处理一致性

### 4. 处理超期要可见

要明确记录：

- 某个 block 是否处理超时
- 是否发生数据覆盖
- 是否出现异常状态

## 可选扩展方向

你后续可以按兴趣扩展：

### 音频方向

- 音频频谱分析
- 简单均衡器
- 音量和频带能量检测

### 振动分析方向

- 加速度传感器采样
- 包络分析
- 主频和谐波分析

### 电力信号方向

- 工频测量
- 谐波分析
- `RMS`、频率、相位估计

## 项目验收标准

项目完成后，至少达到以下标准：

- 连续运行稳定
- 采样率明确且可验证
- 基础统计量可靠
- `FFT` 主峰检测有效
- 性能统计可见
- 文档能说明系统架构和关键设计取舍

## 完成后你应该具备的能力

做完这个项目后，你对 `STM32H743` 的掌握应当不再停留在“会配外设”，而是具备：

- 设计高速采样链路的能力
- 解决 `DMA + Cache` 问题的能力
- 实现基础实时信号处理算法的能力
- 把算法做成稳定工程的能力
