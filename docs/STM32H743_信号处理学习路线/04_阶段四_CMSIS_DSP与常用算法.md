# 第四阶段：CMSIS-DSP 与常用算法

## 学习目标

这一阶段开始从“数据搬运”进入“数据处理”。目标不是一口气学完整个数字信号处理理论，而是围绕嵌入式实时处理的常见需求建立可落地能力。

建议你优先建立以下能力：

- 会使用 `CMSIS-DSP`
- 能实现常见时域处理
- 能做基础频域分析
- 能评估算法耗时和精度

## 为什么先学这些算法

对嵌入式信号处理项目来说，最常见的第一批需求通常是：

- 去直流
- 平滑
- 降噪
- 峰值或包络检测
- 频谱分析
- 特征提取

这意味着你应当先掌握这些算法模块：

- 均值和方差
- 滑动平均
- `FIR`
- `IIR`
- `FFT`
- 幅值、能量、均方根

## 推荐学习顺序

### 1. 先做简单统计和时域特征

先实现这些：

- 均值
- 最大最小值
- 均方值
- `RMS`
- 峰峰值

意义：

- 实现简单
- 容易验证
- 能帮你确认采样链路是否正常

### 2. 再做滑动平均和简单 FIR

学习重点：

- 窗口长度如何影响平滑效果和延迟
- `FIR` 系数的含义
- 卷积的基本代价

你要理解的是“滤波效果”和“实时开销”之间的 tradeoff。

### 3. 学 IIR

学习重点：

- 一阶和二阶滤波器
- 直接型结构
- 系数量化和稳定性

这里不必追求复杂设计，先掌握如何在单片机上稳定运行。

### 4. 学 FFT

这是信号处理项目中最常见的进阶目标。学习重点：

- 采样率、点数、频率分辨率关系
- 窗函数的作用
- 频谱泄漏
- 实数 FFT 的使用方式
- 幅值归一化

### 5. 建立“采样块处理”的思维

在嵌入式里，很多算法不是逐点无限流式处理，而是：

- 收到半缓冲
- 对一个 block 做处理
- 输出结果

这和 `DMA`、双缓冲、实时任务划分高度相关。

## 建议实验

### 实验 1：实时统计量输出

内容：

- 对每个半缓冲数据计算均值、最大值、`RMS`
- 串口打印或发送到上位机

目标：

- 建立最基础的数据分析能力

### 实验 2：滑动平均滤波

内容：

- 设计不同窗口长度
- 对比噪声抑制和响应延迟

目标：

- 理解简单滤波为何有用，也理解它为什么不总是合适

### 实验 3：FIR 或 IIR 低通滤波

内容：

- 对一个带噪输入做低通处理
- 比较输入输出波形

目标：

- 学会把“算法公式”变成嵌入式里的数据流处理

### 实验 4：FFT 频谱分析

内容：

- 固定采样率采集一段数据
- 做窗口处理
- 运行 `FFT`
- 找主峰频率

目标：

- 跑通“采样 -> 窗函数 -> FFT -> 频谱特征输出”闭环

## CMSIS-DSP 学习建议

不要把 `CMSIS-DSP` 当成黑盒。建议每学一个函数都回答：

- 输入输出格式是什么
- 数据类型是 `float`、`q15` 还是 `q31`
- 状态变量和系数如何组织
- 这个函数是逐点处理还是块处理
- 它的计算量大概多大

## 数据类型选择建议

### 浮点

适合：

- 学习阶段
- 原型验证
- 对开发效率要求高

优势：

- 代码直观
- 调参方便

注意：

- 仍然要测算法耗时

### 定点

适合：

- 对性能、存储和确定性要求更高的场景

注意：

- 溢出
- 缩放
- 系数量化

对你当前阶段，建议先用浮点把链路跑通，再考虑定点优化。

## 常见误区

- 看到 `CMSIS-DSP` 有函数就直接调，不理解输入输出约束
- 只关注滤波效果，不测执行时间
- 只看算法公式，不考虑块长和缓冲策略
- FFT 跑出来就信，不验证频率分辨率和幅值缩放

## 阶段验收标准

完成本阶段后，你应该能做到：

- 对采样块计算基础统计量
- 实现一个稳定运行的简单滤波器
- 使用 `CMSIS-DSP` 完成一次频谱分析
- 说明采样率、点数、频率分辨率的关系
- 对算法处理时间做基本测量
