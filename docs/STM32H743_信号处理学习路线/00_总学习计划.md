# STM32H743 信号处理学习总计划

## 适用对象

这套路线面向已经学过 `STM32F4`，准备转向 `STM32H743` 并用于信号采集、滤波、频谱分析和实时处理的开发者。

你的重点不是从头重学单片机，而是把已有的 `F4` 经验迁移到 `H743` 的高性能系统结构上。对信号处理来说，真正决定上限的不是 GPIO 和串口，而是以下几个点：

- 采样链路是否稳定
- `DMA` 与 `Cache` 是否协同正确
- 数据缓冲是否放在合适的内存区
- 算法吞吐能否满足实时性
- 中断、任务、数据搬运链路是否清晰

## 建议学习顺序

按下面顺序推进，不建议跳阶段：

1. 第一阶段：理解 `H743` 和 `F4` 的核心差异
2. 第二阶段：建立可靠的采样链路
3. 第三阶段：解决 `内存 + Cache + DMA + MPU` 这组系统问题
4. 第四阶段：进入 `CMSIS-DSP` 和常用信号处理算法
5. 第五阶段：补足实时系统与工程化能力
6. 第六阶段：完成一个完整的信号处理项目

## 总体周期

按每周投入 `5` 到 `8` 小时估算，建议用 `6` 到 `8` 周完成。

### 第 1 周

- 看懂 `H743` 的时钟树、存储结构和总线域
- 完成最小系统、串口打印、定时器基础

### 第 2 周

- 完成 `TIM` 触发 `ADC`
- 建立 `ADC + DMA` 的连续采样链路
- 开始记录采样率、缓冲长度、CPU 占用

### 第 3 周

- 学习 `Cache`、`MPU`、内存区放置
- 解决 `DMA` 数据一致性问题
- 做双缓冲或 Ping-Pong Buffer 实验

### 第 4 周

- 进入 `CMSIS-DSP`
- 实现均值、滑动平均、FIR、IIR、FFT
- 形成“采样 -> 缓冲 -> 算法 -> 输出”的最小闭环

### 第 5 周

- 引入 `FreeRTOS` 或至少建立清晰的任务边界
- 评估实时性、延迟、抖动
- 学习调试和性能测量方法

### 第 6 到 8 周

- 做综合项目
- 完成稳定性验证
- 补文档、沉淀模板工程

## 每阶段产出要求

每完成一个阶段，都至少留下以下产出：

- 一份自己的学习笔记
- 一份能稳定运行的工程
- 一组可复现实验
- 一份阶段验收记录

## 学习过程中的判断标准

判断你是否真正学会，标准不是“例程跑起来了”，而是你是否能回答这些问题：

- 采样频率由谁决定，误差从哪里来
- `DMA buffer` 为什么放在某块内存而不是另一块
- 打开 `D-Cache` 后为什么数据会错
- 某个滤波算法为什么选定点或浮点实现
- 当前处理链路的实时瓶颈在采样、中断、搬运还是算法

## 使用建议

- 每个阶段先看文档中的“必须搞懂”
- 再做“建议实验”
- 最后对照“验收标准”

不要把 `H743` 学成“更快的 F4”。对信号处理场景来说，`H743` 更像一个需要你主动设计数据路径和内存布局的小系统。
