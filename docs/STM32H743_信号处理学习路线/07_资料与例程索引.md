# 资料与例程索引

## 官方资料优先级

学习 `STM32H743` 时，建议按以下优先级使用资料：

1. `Datasheet`
2. `Reference Manual`
3. `Application Note`
4. `CubeH7` 官方例程
5. 第三方教程

原因很直接：`H743` 的很多关键问题都和系统结构、内存、缓存、一致性有关，第三方教程很容易只讲“怎么配”，不讲“为什么这样配”。

## 每阶段建议重点查阅的资料

### 阶段一：系统架构

重点查：

- `H743` 数据手册中的器件概览
- 参考手册中的系统架构
- 时钟树
- 存储映射

你要输出：

- 一张自己的内存和数据路径简图

### 阶段二：采样链路

重点查：

- `TIM`
- `ADC`
- `DMA`
- 中断与事件

你要输出：

- 一条 `TIM -> ADC -> DMA -> Buffer` 时序图

### 阶段三：系统问题

重点查：

- `Cache`
- `MPU`
- 内存属性
- 与 `DMA` 相关的应用笔记

你要输出：

- 一份 `DMA buffer` 放置和缓存策略说明

### 阶段四：算法

重点查：

- `CMSIS-DSP` 文档
- `FFT`
- 滤波器基础
- 窗函数基础

你要输出：

- 一份算法模块输入输出说明

### 阶段五：工程化

重点查：

- `FreeRTOS` 基础任务模型
- 官方实时系统示例
- 调试和性能测量方法

你要输出：

- 一份任务划分图和性能统计表

## 官方例程使用方法

不要只看例程“能不能跑”，建议按下面方式使用：

1. 先运行官方例程
2. 再画出关键数据路径
3. 再改一个关键参数
4. 再记录变化
5. 最后把例程中的关键配置迁移到你自己的模板工程

## 学习时必须关注的关键词

以后你在看手册、例程、应用笔记时，看到这些关键词要特别敏感：

- `trigger`
- `conversion time`
- `DMA circular`
- `half transfer`
- `cache coherence`
- `MPU attributes`
- `memory region`
- `throughput`
- `latency`
- `jitter`

## 阶段自检清单

每推进一阶段，都用下面的问题自检：

- 我是否知道当前数据从哪里来，到哪里去
- 我是否能量化当前链路的采样率和处理时间
- 我是否知道当前缓冲区放在哪，为什么放在那
- 我是否知道当前结果错了该先查哪一层
- 我是否把结论写成了文档，而不是只留在脑子里
