# 第二阶段：采样链路与基础外设

## 学习目标

这一阶段的目标是建立一条稳定、可测量、可扩展的采样数据链路，而不是把所有外设都刷一遍。

对信号处理来说，优先级应当是：

1. `TIM`
2. `ADC`
3. `DMA`
4. 数据输出链路

只有这条链路稳定了，后面的滤波、FFT、判决算法才有意义。

## 这一阶段要解决什么问题

你需要把以下问题彻底搞清楚：

- 采样频率是谁决定的
- 采样是否均匀
- 数据如何搬进内存
- 缓冲区什么时候“满”
- 处理代码在什么时刻介入

## 必须搞懂的内容

### 1. 定时器触发采样

信号处理不要依赖软件轮询触发 `ADC`。你要优先建立：

- `TIM` 周期事件
- `TIM TRGO`
- `ADC` 外部触发

因为这样更容易得到稳定的采样周期。

### 2. ADC 连续采样与通道组织

学习重点：

- 单通道固定频率采样
- 多通道扫描
- 采样时间与源阻抗关系
- 分辨率、对齐方式、转换时间

这里先不要急着追高采样率，先保证采样行为可解释。

### 3. DMA 传输模式

你需要掌握：

- 普通模式
- 循环模式
- 半传输中断
- 传输完成中断

对于后续信号处理，最关键的是半缓冲和满缓冲的处理方式。

### 4. 缓冲区设计

先建立最基础的两种方案：

- 单缓冲
- 双缓冲或 Ping-Pong Buffer

你要形成的意识是：算法处理不能阻塞采样链路。

### 5. 数据输出与观测

建议至少准备一种观测方式：

- 串口输出少量统计信息
- `SWV` 或调试器观察缓冲区
- 导出数据到上位机画波形

如果看不到数据，就很难判断采样链路到底有没有问题。

## 推荐实验顺序

### 实验 1：定时器驱动的单通道采样

内容：

- 使用 `TIM` 产生固定频率触发
- 使用 `ADC` 采一个通道
- 使用 `DMA` 搬运到数组

你要观察：

- 采样率是否符合预期
- 缓冲区长度变化是否符合预期
- 输出波形是否连续

### 实验 2：半缓冲和满缓冲处理

内容：

- 开启 `DMA` 半传输和完成回调
- 在两个回调里分别做最轻量的统计

你要观察：

- 回调触发间隔是否稳定
- 某一半缓冲处理时间是否会侵占下一次采样

### 实验 3：双通道或多通道采样

内容：

- 尝试两个模拟输入
- 观察交织后的数据排列
- 写出解析函数

你要搞清楚：

- 数据在内存中的顺序
- 每个样本点对应哪个通道

### 实验 4：构建最小处理闭环

内容：

- 每收到半缓冲数据，计算均值、最大值、均方值中的一种
- 把结果用串口或上位机输出

这个实验非常重要，因为它标志着你从“采样”进入“实时处理”。

## 阶段内建议顺序

建议按下面顺序推进：

1. `GPIO + UART` 只做最小辅助
2. `TIM` 产生精确节拍
3. `ADC` 单通道采样
4. `DMA` 循环搬运
5. 半缓冲回调处理
6. 双缓冲或多通道采样
7. 接上最简单的统计或滤波

## 常见坑

- 用软件触发代替定时器触发，导致采样不均匀
- 只关注 `ADC` 配置，不看定时器频率
- 回调中做太重的处理，破坏实时性
- 看见数据能出来，就默认采样率准确
- 没有记录缓冲区长度和中断周期

## 阶段验收标准

完成本阶段后，你应该已经具备以下能力：

- 能独立建立 `TIM -> ADC -> DMA -> Buffer` 链路
- 能计算理论采样率并和实测值比较
- 能处理半缓冲和满缓冲事件
- 能对一段采样数据做最简单的实时统计
- 能解释当前系统的采样节拍和数据搬运机制
