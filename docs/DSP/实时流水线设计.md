# 实时信号处理流水线设计

工程已具备 ADC 周期采样、CMSIS-DSP 集成、FFT/FIR/IIR 三套基础能力。本文档说明把它们串成一条 **ADC → 滤波 → FFT** 流水线的设计方案、数据流、状态机和 CLI 接口。

## 1. 目标

提供一个可由 CLI 控制的实时管道，让用户：

- 选择是否启用滤波（`none` / FIR / IIR Biquad）
- 选择 FFT 长度（256 / 512 / 1024）
- 单次触发或持续运行，把结果（峰值 bin、频率、幅度）通过 USART1 输出
- 切换滤波器时自动清状态，避免污染

## 2. 数据流

```
TIM2 触发 ──► ADC1 (DMA 双半缓冲) ──► s_adc_dma_buffer[2*128]
                                          │
                       ADC ISR (HAL_ADC_ConvHalfCpltCallback / FullCpltCallback)
                                          │
                                          ▼
                          ADC_APP_QueueBlock 入块队列（已有功能）
                                          │
                                          ▼
                       ┌──────────────────────────────────────────────┐
                       │  scheduler 任务 dsp_pipeline_proc (1 ms)     │
                       ├──────────────────────────────────────────────┤
                       │  if (pipeline 未启用) return                 │
                       │  while ADC_APP_PopBlock 拿到一块 (uint16, 256)│
                       │      ├─ uint16 → float32                     │
                       │      │     可选去 DC：sample - 2048           │
                       │      ├─ 过滤波器（none/FIR/Biquad，按枚举）  │
                       │      ├─ 累积到 s_acc_buf[fft_len]            │
                       │      └─ 累积满：触发 FFT                     │
                       │              ├─ RFFT + cmplx_mag             │
                       │              ├─ FindPeak                     │
                       │              ├─ 输出：peak_bin/freq/mag      │
                       │              ├─ 累积器清空                   │
                       │              └─ 若 mode=ONESHOT，进入 IDLE   │
                       └──────────────────────────────────────────────┘
```

数据宽度变化：

```
ADC raw uint16 [0..65535]      ← 注意：当前 ADC 配置为 16-bit 分辨率（adc.c）
   │
   │  (粗去 DC) raw - 32768.0f （PL_ADC_MIDSCALE）
   ▼
float32 中心化样本（典型范围 [-32768, +32767]）
   │
   │  滤波（FIR/Biquad/旁路）
   ▼
float32 滤波后样本
   │
   │  累积满 fft_len 时
   ▼
(精确去 DC) 减帧均值 arm_mean_f32
   │
   ▼
float32 RFFT + 取模 → mag[fft_len/2]
```

去 DC 分两级：
1. **粗去 DC（按块）**：减 ADC 中点 32768，把数值拉回零附近，避免巨大 DC
   吃掉滤波器的浮点动态范围，也让 IIR 启动瞬态变小。
2. **精确去 DC（按帧）**：FFT 前用 `arm_mean_f32` 算累积器的均值再减，
   消除残余 DC，防止泄漏到 bin 1。

不做 `/65536` 归一化，FFT 幅度数量级跟随 raw，便于人眼对比。

## 3. 模块边界

新增 `App/dsp/dsp_pipeline.{h,c}`，与现有模块的依赖关系：

```
dsp_pipeline.c
    │
    ├── 上游消费：ADC_APP_PopBlock, ADC_APP_GetSampleRateHz, ADC_APP_BLOCK_SAMPLES
    │
    ├── 算法依赖：DSP_RFFT_Magnitude, DSP_FindPeak,
    │             DSP_FIR_F32_*, DSP_Biquad_F32_*
    │
    ├── 系数来源：dsp_filters.h（不直接用 dsp_filters.c 的 state，自带 state buffer）
    │
    └── 输出：my_printf 到指定 UART
```

`dsp_pipeline` 不持有 ADC 配置权（不改 ADC 采样率 / block 长度），只**消费** ADC 已经产出的块。这样耦合最低。

## 4. 关键状态

```c
typedef enum {
    PIPELINE_MODE_IDLE = 0,
    PIPELINE_MODE_ONESHOT,   // 跑一次后回 IDLE
    PIPELINE_MODE_STREAM,    // 持续跑直到 stream off
} pipeline_mode_t;

typedef enum {
    PIPELINE_FILTER_NONE = 0,
    PIPELINE_FILTER_FIR_MA5,
    PIPELINE_FILTER_FIR_TEMPLATE,
    PIPELINE_FILTER_BIQUAD_LP_2K,
    PIPELINE_FILTER_BIQUAD_TEMPLATE,
} pipeline_filter_t;

// 模块单例状态
static struct {
    pipeline_mode_t   mode;
    pipeline_filter_t filter;
    dsp_fft_size_t    fft_size;       // 256 / 512 / 1024
    uint8_t           dc_remove;      // 0 / 1
    uint16_t          acc_count;      // 已累积样本数 [0, fft_size]
    uint32_t          frame_seq;      // 第几帧
    UART_HandleTypeDef *out_huart;
} s_pl;
```

状态切换：

```
IDLE ──pipeline run────► ONESHOT
IDLE ──pipeline stream on──► STREAM
ONESHOT ──完成一帧──► IDLE
STREAM ──pipeline stream off──► IDLE

任何状态：
   pipeline filter <name>     → 切换 + 清 state + acc_count = 0
   pipeline fft <size>        → 切换 + acc_count = 0
   pipeline dc on|off         → 切换 + acc_count = 0
```

## 5. 缓冲区清单

```
s_acc_buf[1024]      累积输入（按最大 FFT 长度分配）   4 KB
s_fft_scratch[1024]  RFFT 复数中间结果                  4 KB
s_fft_mag[512]       幅度谱                             2 KB
s_fir_state[N+127]   FIR state（N=最大 num_taps）       约 1 KB
s_biq_state[2*M]     Biquad state（M=最大 stages）      < 1 KB
─────────────────────────────────────────────
合计                                                    ~12 KB
```

均为 static 全局，放 .bss。RAM_D1（AXI SRAM）即可。

## 6. CLI 接口

```
pipeline status                 显示当前 mode / filter / fft / dc / frame_seq
pipeline filter <name>          切滤波：none / fir_ma5 / fir_template / biquad_butter2_2k / biquad_template
pipeline filter ?               列出可选名
pipeline fft <256|512|1024>     设 FFT 长度
pipeline fft ?                  显示当前
pipeline dc on|off|?            去 DC 开关
pipeline run                    单次运行
pipeline stream on|off          持续运行
```

输出格式（每帧一行，可被脚本解析）：

```
[PIPELINE] seq=12 filter=fir_ma5 fft=1024 fs=256000 Hz peak_bin=128 peak_freq=32000.00 Hz mag=243.51
```

## 7. 异常与边界

| 情况 | 处理 |
|---|---|
| ADC 未启动 | `pipeline run/stream on` 报错，不进入工作态 |
| 用户切 `fft <size>` 当前正在累积 | 丢弃当前累积，从下一帧开始攒 |
| 用户切滤波器在 STREAM 模式 | 清 filter state，不退出 STREAM，下一帧用新滤波 |
| ADC 块队列被填爆（处理跟不上）| 由 ADC_APP 已有的 dropped_block 计数器统计，pipeline 不主动恢复 |
| FFT 长度未就绪（DSP_Init 没跑）| `DSP_Init()` 在 main 启动时已调用，正常情况不会触发 |

## 8. 性能预算

@ 480 MHz CM7 + FPv5-D16 + ICache，DCache 关：

| 步骤 | 耗时（每 ADC 块 128 样本）|
|---|---|
| uint16→float32 + DC 减法 | ~3 µs |
| FIR 5-tap | ~5 µs |
| Biquad 1 段 | ~3 µs |
| 复制累积 | ~2 µs |

@ 累积满 1024 一次：

| 步骤 | 耗时 |
|---|---|
| 1024-pt RFFT_F32 | ~50–100 µs |
| arm_cmplx_mag_f32 (512 点) | ~10 µs |
| arm_max_f32 (512 点) | ~5 µs |
| my_printf 输出 | ~200 µs（取决于 UART 波特率）|

每帧 4 ms 预算下，纯计算 < 200 µs，余量大。

UART 输出**已改为 DMA 异步**（`my_printf` 推进环形缓冲，`HAL_UART_Transmit_DMA` 后台发），调用方零阻塞。波特率从 115200 提到 921600 后，130 字节/行 × 11.3 µs/字节 ≈ 1.5 ms，仍可能不够 250 帧/秒（需 4 ms/行）。这种情况下可用 `pipeline rate <N>` 降频输出（每 N 帧打一次），或用 `uarttx` CLI 命令观察 `dropped` 计数判断是否要进一步压缩格式。

## 9. 与 cache 的关系

工程当前 ICache 开、DCache 关。该流水线**完全兼容当前配置**，不需要任何 cache/MPU 改动。

未来若启用 DCache：
- ADC DMA 缓冲（`.dma_buffer` → RAM_D2）必须配 MPU 设为非 cacheable，否则 CPU 拿到旧数据
- pipeline 的 `s_acc_buf` 等内部 buffer 在 RAM_D1，可继续 cacheable，无问题

详见 `docs/STM32H7_DMA缓冲区与链接脚本说明.md`。

## 10. 扩展指引：加新滤波器

1. 按 `docs/DSP/MATLAB滤波器设计与导出.md` 在 MATLAB 设计、生成系数
2. 在 `App/dsp/dsp_filters.h` / `.c` 加常量
3. 在 `dsp_pipeline.h` 的 `pipeline_filter_t` 枚举末尾加一项
4. 在 `dsp_pipeline.c::pipeline_apply_filter` 的 switch 加一条 case
5. 在 CLI handler 的名字解析里加一条 `strcmp` 分支
6. 重编译，CLI `pipeline filter ?` 自动列出新名字

## 11. 已知简化（先不做的事）

- 没有 overlap：每次 FFT 之间不复用样本，时间分辨率 = fft_len / fs
- 没有窗函数：直接矩形窗 RFFT，频谱泄漏在所难免（纯正弦不影响找峰）
- 没有归一化：保留 raw 数量级，便于人工对比 ADC 实际幅度
- 没有自动增益：超出 12-bit 范围的输入会饱和，由 ADC 硬件层面承担
- 没有同时输出多个频谱：只输出峰值 bin（如需全谱后续可加 `pipeline dump`）

## 12. 测试路径

| 验证项 | 方法 |
|---|---|
| Pipeline 通路 | DAC 输出 1 kHz 正弦 → ADC 采集 → `pipeline run` 应得到对应 bin |
| 滤波切换正确 | DAC 输 5 kHz、FFT_LEN=1024、fs=256k → 切 `biquad_lp_2k` 应看到 mag 大幅下降 |
| 状态清零正确 | 切滤波后立即 `run`，输出无 NaN/Inf，前几个样本不会异常大 |
| 流式稳定 | `stream on` 持续运行 1 分钟，无 dropped_block 急剧上升 |
| CLI 解析鲁棒性 | 错参数（`pipeline fft 999` / `pipeline filter xxx`）应给提示，不崩 |
