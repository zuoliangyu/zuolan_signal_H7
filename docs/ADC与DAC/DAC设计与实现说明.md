# DAC 设计与实现说明

本文档用于规划当前工程中 `DAC` 部分的设计、配置、实现路径和调试重点。当前平台为 `STM32H743`。

## 1. 当前目标与实现状态

当前按“先静态输出、再动态输出”的顺序推进，不一开始就叠加波形表、定时器触发和 DMA。

当前已经完成第一阶段最小实现：

- 使用 `DAC1_CH1`
- 输出引脚为 `PA4`
- 触发方式为 `TIM6 TRGO`
- 直流输出由应用层软件设置 `offset`
- 波形输出由 `TIM6 + DAC DMA Circular` 驱动
- 默认上电输出值为 `1650mV`

建议目标顺序：

1. 跑通静态电压输出
2. 跑通软件更新输出值
3. 再决定是否加入定时器触发
4. 再决定是否加入 `DMA` 连续波形输出

这样做的原因：

- `DAC` 本身链路比 `ADC` 更直观，适合先拿来验证模拟输出基础
- 如果一开始就做波形输出，定时器、DMA、缓冲区三个变量会同时叠加

## 2. 推荐实现路线

### 第一阶段：单通道静态输出

适用场景：

- 先验证 `DAC` 通道、引脚和电压输出范围

建议：

- 先只开一个通道
- 通过软件直接设置输出值
- 用万用表或示波器确认输出

当前落地结果：

- `CubeMX` 已生成 `Core/Src/dac.c` 和 `Core/Inc/dac.h`
- 应用层已新增：
  - `App/dac/dac_app.c`
  - `App/dac/dac_app.h`
- `main.c` 中只做模块装配，实际 DAC 控制收敛到 `App/dac`

目标：

- 先证明“输出链路是通的”
- 先证明“应用层可以独立管理 DAC 输出值”

### 第二阶段：软件更新输出值

适用场景：

- 需要根据命令或程序逻辑改变输出电压

建议：

- 在 `App/dac` 中提供设置接口
- 输出值管理放应用层，不直接散落在 `main.c`

目标：

- 先证明“上层逻辑能可靠控制 DAC”

### 第三阶段：定时器触发更新

适用场景：

- 需要固定速率刷新输出

建议：

- 用一个专门定时器做触发
- 明确定义输出刷新频率

目标：

- 先证明“更新节拍是稳定的”

### 第四阶段：DMA 连续波形输出

适用场景：

- 需要输出简单波形
- 需要减少 CPU 参与

建议：

- 在前面阶段稳定后再加
- 明确波形缓冲区长度、数据宽度和刷新频率

目标：

- 先证明“波形连续输出没有断裂和抖动”

## 3. `STM32H743` 上的默认建议

如果当前还没有明确业务约束，默认建议如下：

- 应用层目录：`App/dac`
- 基础版本：单通道静态输出
- 后续增强：定时器触发 + DMA 波形输出

需要特别注意：

- 输出脚位必须先和板级实际连接核对
- 如果引入 `DMA`，同样要考虑缓冲区内存与后续 `D-Cache`
- 如果要和 `ADC` 做联动，优先先分别跑通，再做同步

## 5. 当前接口说明

当前 `App/dac` 提供的接口如下：

- `DAC_APP_Init()`
  - 初始化默认参数并启动 `DAC1_CH1`
- `DAC_APP_Start()`
  - 按当前保存参数启动 DAC 输出
- `DAC_APP_Stop()`
  - 停止当前 DAC 输出，但保留当前参数
- `DAC_APP_SetMode(dac_app_mode_t mode)`
  - 设置当前输出模式
- `DAC_APP_SetAmpMv(uint16_t amp_mv)`
  - 设置当前振幅
- `DAC_APP_SetOffsetMv(uint16_t offset_mv)`
  - 设置当前偏置
- `DAC_APP_SetFreqHz(uint32_t frequency_hz)`
  - 设置当前频率
- `DAC_APP_SetDutyPercent(uint8_t duty_percent)`
  - 设置当前方波占空比百分比
- `DAC_APP_IsStarted()`
  - 获取当前 DAC 输出状态
- `DAC_APP_GetModeString()`
  - 获取当前模式字符串
- `DAC_APP_GetCurrentRaw()`
  - 获取当前用于输出的原始值基准

模式解释：

- `dc`
  - 输出固定电压
  - 实际输出值取 `offset_mv`
- `sine`
  - 以 `offset_mv` 为中心输出正弦波
  - 振幅由 `amp_mv` 决定
- `tri`
  - 以 `offset_mv` 为中心输出三角波
  - 振幅由 `amp_mv` 决定
- `square`
  - 以 `offset_mv` 为中心输出方波
  - 振幅由 `amp_mv` 决定
  - 占空比由 `duty_percent` 决定

最终输出电压取决于板上的 `VDDA` 实际值。

当前参数模型：

- `mode`
  - `dc / sine / tri / square`
- `amp_mv`
  - 当前振幅，单位 `mV`
- `offset_mv`
  - 当前偏置，单位 `mV`
- `freq_hz`
  - 当前频率，单位 `Hz`
- `duty_percent`
  - 当前方波占空比，单位百分比

所有参数都独立保存。

当 DAC 处于运行状态时：

- 修改任一参数都会立即生效
- 修改一个参数不会重置其他参数

当 DAC 已停止时：

- 可以继续修改参数
- 新参数会在下次 `dac start` 时统一生效

当前波形实现约定：

- 波形缓冲区长度固定为 `128` 点
- 波表放在 `.dma_buffer`，由链接脚本映射到 `RAM_D2`
- 电压换算固定按参考值 `3300mV` 计算
- 当 `offset +/- amp` 超出 `0..3300mV` 时，输出会被裁剪到合法范围

## 7. 风险

最主要风险如下：

- 板级输出引脚与预期通道不一致
- 参考电压理解错误，导致电压换算判断出错
- 定时器触发频率和预期不一致
- `DMA` 波形输出时缓冲区组织错误
- 后续启用 `D-Cache` 后出现输出数据不连续

当前还需要注意：

- 如果后续要做高速更新或 DMA 输出，需要重新评估缓冲区内存区域
- 如果后续引入 `D-Cache`，DMA 方案需要单独处理一致性
- 当前默认值为 `2048`，如果业务要求上电默认 `0V`，可以直接调整 `DAC_APP_DEFAULT_RAW_VALUE`

## 8. 验收标准

后续开始实现 `DAC` 时，建议至少满足：

- 静态输出值可控，且电压变化符合预期
- 上层接口可以稳定修改输出值
- 如果加入定时器触发，刷新频率清晰可控
- 如果加入 `DMA`，波形输出稳定
- 应用逻辑不堆进 `main.c`

当前最小实现的验收标准：

- `PA4` 可测得稳定直流电压
- 修改 `DAC_APP_SetValueRaw()` 传入值后，输出电压能同步变化
- `main.c` 中不直接散落 `HAL_DAC_*` 业务调用

## 9. 调试建议

后续调试建议顺序：

1. 先确认引脚和输出通道
2. 再确认静态输出值
3. 再确认上层接口控制
4. 最后再调定时器和 `DMA`
