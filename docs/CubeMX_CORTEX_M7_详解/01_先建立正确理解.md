# 01 先建立正确理解

## 1. 先把四件事分开

很多人一打开 `CubeMX -> CORTEX_M7` 页面就晕，是因为把四类问题混在一起了：

1. 性能问题
   - `Speculation`
   - `ICache`
   - `DCache`
2. 访问权限问题
   - `MPU`
   - `Region Access Permission`
   - `Instruction Access`
3. 一致性问题
   - `CPU` 和 `DMA` 是否看到同一份数据
4. 工程落地问题
   - 数组放在哪
   - 链接脚本怎么配
   - RTOS 任务和 DMA buffer 怎么分区

如果你不先把这四类分开，后面看任何一个选项都会觉得“好像都有关，又不知道到底在管什么”。

## 2. 对 H743 来说最核心的现实

`STM32H743` 是 Cortex-M7 内核，性能高，但它的复杂度也比 F4 高不少。ST 官方资料里明确说明 H7 有：

- L1 Cache
- TCM RAM
- 多块 SRAM
- MPU

这意味着你不能再用 F4 那种“全局数组随便放，DMA 跑起来就行”的思路长期做大工程。

官方参考：

- [STM32H743/753 产品页](https://www.st.com/en/microcontrollers/stm32h7x3.html)
- [PM0253](https://www.st.com/resource/en/programming_manual/pm0253-stm32f7-series-and-stm32h7-series-cortexm7-processor-programming-manual-stmicroelectronics.pdf)

## 3. ICache 和 DCache 不是一回事

### ICache

它缓存的是“指令”。

你可以把它理解成：

- CPU 取代码更快
- 循环执行更快
- 函数调用和跳转时更容易受益

通常它带来的副作用很少，所以在多数项目里是“建议尽早开启”的。

### DCache

它缓存的是“数据”。

你可以把它理解成：

- 数组访问更快
- 算法中间变量读写更快
- 但 CPU 看到的数据，未必和内存里实时一致

这里就会直接撞上 `DMA 一致性` 问题。

## 4. 为什么 DCache 容易把人坑住

因为 `DMA` 一般绕过 CPU 的 Cache，直接跟内存打交道。

于是会出现两种常见错觉：

### 错觉 1

“ADC DMA 明明已经把新数据搬进 buffer 了，为什么 CPU 读到的是旧值？”

原因通常是：

- DMA 写的是内存
- CPU 读的是 DCache 里的旧副本

### 错觉 2

“我明明改了发送缓冲区，为什么 UART/SPI DMA 发出去还是旧值？”

原因通常是：

- CPU 改的是 cache 行
- 数据还没 clean 回内存
- DMA 拿到的还是旧内存内容

这就是为什么 ST 在 [AN4839](https://www.st.com/resource/en/application_note/an4839-level-1-cache-on-stm32f7-series-and-stm32h7-series-stmicroelectronics.pdf) 里专门讲 cache coherence。

## 5. MPU 不是“高级可选项”，而是“内存规则工具”

很多人一听 `MPU` 就以为它只是做安全保护，其实在 H7 项目里它非常工程化。

你可以把 MPU 理解成：

- 给某段地址空间贴标签
- 规定这块区域能不能执行
- 能不能读写
- 能不能缓存
- 适不适合做 DMA 共享区

所以 MPU 的工程价值主要是：

1. 给 DMA buffer 建 non-cacheable 区域
2. 给某些区域禁止执行
3. 给不同 SRAM 区制定不同访问属性

## 6. 多任务不是核心难点，组合问题才是

你前面提到后面会做多任务。这里要先纠正一个很关键的理解：

- 不是“因为多任务，所以必须先开 Cache”
- 而是“当你进入多任务 + DMA + 大缓冲 + 算法处理后，必须把 Cache/MPU 策略想清楚”

真正危险的组合是：

- RTOS
- ADC/SPI/UART/ETH/SDMMC 的 DMA
- 环形缓冲
- DCache
- 大块数组处理

## 7. 当前阶段最重要的认知结论

如果你现在还在建立基础工程，正确顺序通常不是“一上来全开”，而是：

1. 先理解页面上的配置项到底管什么
2. 先开 `ICache`
3. 暂时别急着开 `DCache`
4. 等第一个 DMA 场景稳定后，再系统设计 `DCache + MPU`

## 8. 一句话记忆版

- `Speculation`：偏性能
- `ICache`：偏性能，通常早开
- `DCache`：偏性能，但会带来一致性问题
- `MPU`：偏规则，用来定义“哪块内存按什么方式访问”
